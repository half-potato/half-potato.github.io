1 # RGBDSLAM<i>v2</i>
2 ... is a state-of-the-art SLAM system for RGB-D cameras, e.g., the Microsoft
3 Kinect or the Asus Xtion Pro Live. You can use it to create 3D point clouds or
4 OctoMaps.
5 
6 RGBDSLAMv2 is based on the open source projects, ROS, OpenCV, OpenGL, PCL,
7 OctoMap, SiftGPU, g2o, and more - Thanks!
8 
9 A journal article with a system description and performance evaluation
10 can be found in the following publication:
11 
12 "3D Mapping with an RGB-D Camera",<br/>
13 *F. Endres, J. Hess, J. Sturm, D. Cremers, W. Burgard*,<br/>
14 IEEE Transactions on Robotics, 2014.
15 
16 Additional information can be found here:<br/>
17 * www.informatik.uni-freiburg.de/~endres
18 * http://www.ros.org/wiki/rgbdslam
19 * http://answers.ros.org/questions/tags:rgbdslam

27 # Installation ################################################################
28 
29 1. Put RGBDSLAMv2 in a catkin workspace: See [the catkin tutorial](http://wiki.ros.org/catkin/Tutorials/create_a_workspace)
30  for details. Use git to clone this repository into your workspace's "src/" directory. Or download RGBDSLAMv2 as an [archive](http://codeload.github.com/felixendres/rgbdslam_v2/zip/indigo) and extract it to "src/".
31 
32 2. Use rosdep (i.e. "rosdep install rgbdslam") to install missing
33  dependencies. For details see http://wiki.ros.org/ROS/Tutorials/rosdep
34 
35 4. To build RGBDSLAMv2 go to your catkin workspace and execute "catkin_make".
36  If you get an error about the missing siftgpu library, execute "catkin_make" again.

64 # Installation done! What's next?
65 See the sections below for more details on the usage.
66 But to get you started quickly here's the most important pointers:
67 
68 - If you want to use RGBDSLAMv2 with an RGB-D camera you may have
69  to install openni (sudo apt-get install ros-indigo-openni-launch).
70 
71 - Check out the launch files in "launch/" for examples and specific
72  use cases. roslaunch rgbdslam openni+rgbdslam.launch is a good starting
73  point for live mapping.
74 
75 - Check out the README in "test/" for running, testing and evaluating
76  RGBDSLAMv2 on Juergen Sturm's RGB-D SLAM Dataset and Benchmark:
77  http://vision.in.tum.de/data/datasets/rgbd-dataset
78  You need cython for the evaluation scripts (sudo apt-get install cython).
79 
80 - If you want to use SURF or SIFT, you will need to build OpenCV from source,
81  including the non-free module (this does not include SIFTGPU, which is
82  included, but needs to be enabled in CMakeLists.txt). In the CMakeLists.txt
83  of RGBDSLAMv2 you can set the build directory of OpenCV and enable the
84  non-free functionality. Note that SIFT and SURF are not the best choice.
85  Due to new (software) features in RGBDSLAMv2, ORB outperforms both.

108 # Usage ##############################################################
109 
110 Most people seem to want the registered point cloud. It is by default sent out
111 on /rgbdslam/batch_clouds when you command RGB-D SLAM to do so (see below). The
112 clouds sent are actually the same as before, but the according transformation -
113 by default from /map to /openni_camera - is sent out on /tf.
114 
115 The octoMap library is compiled into the rgbdslam node. This allows to create
116 the octomap directly. In the GUI this can be done by selecting "Save Octomap"
117 from the "Data" Menu. Online octomapping is possible, but not recommended.
118

121 ## Usage with GUI #################################################################
122 
123 To start RGBDSLAMv2 launch, e.g.,
124 $ roslaunch rgbdslam openni+rgbdslam.launch
125 
126 Alternatively you can start the openni nodes and RGBDSLAMv2 separately, e.g.:
127 roslaunch openni_camera openni_node.launch
128 roslaunch rgbdslam rgbdslam.launch
129 
130 To capture models either press space to start recording a continuous stream or
131 press enter to record a single frame. To reduce data redundancy, sequential
132 frames from (almost) the same position are not included in the final model.
133 
134 Parameters
135 RGBDSLAMv2 is customizable by parameters. These should be set in the launch
136 file. Parameters can be changed during operation from the GUI, however,
137 changes from the GUI may have no effect for many parameters.
138 
139 Visualization
140 The 3D visualization shows the globally optimized model (you might have
141 to click into it to update the view after optimization). Neighbouring
142 points can be triangulated except at missing values and depth jumps. With
143 the shortcut "t", triangulation can be toggled. Since raw points render
144 slightly faster the parameter "cloud_display_type" controls whether
145 triangulation is computed at all - at the time the cloud is received.
146 The parameter "gl_point_size" may be useful to most users.

149 ## Usage without GUI ##############################################################
150 
151 The RosUI is an alternative to the Grapical_UI to run the rgbdslam headless,
152 for example on the PR2. rgbdslam can then be used via service-calls.
153 The possible calls are:
154  * /rgbdslam/ros_ui {reset, quick_save, send_all, delete_frame, optimize, reload_config, save_trajectory}
155  * /rgbdslam/ros_ui_b {pause, record} {true, false}
156  * /rgbdslam/ros_ui_f {set_max} {float}
157  * /rgbdslam/ros_ui_s {save_octomap, save_cloud, save_g2o_graph, save_trajectory, save_features, save_individual} {filename}
158 
159 To start the rgbdslam headless use the headless.launch:
160 $ roslaunch rgbdslam headless.launch
161 
162 Capture single frames via:
163 $ rosservice call /rgbdslam/ros_ui frame
164 
165 Capture a stream of data:
166 $ rosservice call /rgbdslam/ros_ui_b pause false
167 
168 Send point clouds with computed transformations (e.g., to rviz or octomap_server):
169 $ rosservice call /rgbdslam/ros_ui send_all
170 
171 Save data using one of the following:
172 
173 All pointclouds in one file quicksave.pcd in rgbdslam/bin-directory:
174 $ rosservice call /rgbdslam/ros_ui_s save_cloud
175 
176 Every pointcloud in its own file in rgbdslam/bin-directory:
177 $ rosservice call /rgbdslam/ros_ui save_individual
178 
179 /rgbdslam/ros_ui:
180 
181  * reset ''resets the graph, delets all nodes (refreshes only when capturing new images)''
182  * frame ''capture one frame from the sensor''
183  * optimize ''trigger graph optimizer''
184  * reload_config ''reloads the paramters from the ROS paramter server''
185  * quick_save ''saves all pointclouds in one file quicksave.pcd in rgbdslam/bin-directory''
186  * send_all ''sends all pointclouds to /rgbdslam/transformed_cloud (can be visualized with rviz)''
187  * delete_frame ''delete the last frame from the graph (refreshes only when capturing new images)''
188 
189 /rgbdslam/ros_ui_b:
190 
191  * pause <flag> ''pauses or resumes the capturing of images''
192  * record <flag> ''pauses or stops the recording of bag-files, can be found in the rgbdslam/bin-directory''
193 
194 /rgbdslam/ros_ui_f:
195 
196  * set_max <float> ''filters out all datapoints further away than this value (in cm, only for saving to files)''
197 
198 /rgbdslam/ros_ui_s:
199 
200  * save_features <filname> ''saves the feature locations and descriptors in a yaml file with the given filename''
201  * save_cloud <filename> ''saves the cloud to the given filename (should end with .ply or .pcd)''
202  * save_individual <fileprefix> ''saves every scan in its own file (appending a suffix to the given prefix)''
203  * save_octomap <filename> ''saves the cloud to the given filename''
204  * save_trajectory <fileprefix> ''saves the sensor trajectory to the file <fileprefix>_estimate.txt''

227 ## GICP and SIFTGPU ###############################################################
228 
229 If there are problems related to the compilation or linking of GICP or SIFTGPU,
230 you can deactivate these features at the top of CMakeLists.txt. You might get
231 even faster GPU features setting the parameter "siftgpu_with_cuda" but you will
232 need to install the proprietary CUDA drivers that may require a NVidia GPU
233 (see http://www.nvidia.com/object/cuda_gpus.html). For installing the
234 development drivers and the CUDA SDK you can use the following tutorial:
235 http://sublimated.wordpress.com/2011/03/25/installing-cuda-4-0-rc-on-ubuntu-10-10-64-bit/
236 or for ubuntu 10.04: http://ubuntuforums.org/showthread.php?t=1625433 (tested
237 on Ubuntu 10.04 x64) To use SiftGPU you should install "libdevil-dev".
238 
239  Additional compiling information can be changed in
240  external/siftgpu/linux/makefile.
241 
242  If you get an error that the siftgpu library is not found, execute "make" manually
243  in the directory external/siftgpu/ and rerun catkin_make.
244 
245  GICP Generalized ICP can be (de)activated for refining the registration. For
246  more information see http://stanford.edu/~avsegal/generalized_icp.html
